accessibility Accessibility Best Practices

✅ Good Alt Text Examples

alt="Sarah Johnson, Senior Developer"
alt="Profile picture of Alex Chen"
alt="Company logo for Acme Corp"

❌ Poor Alt Text Examples

alt="image" Too generic, not helpful
alt="avatar" Describes component, not content
alt="" Empty alt text (missing required attribute)
alt="photo-1494790108755.jpg" Filename instead of description

Key Accessibility Points:

  • Required: The alt attribute is mandatory for screen readers
  • Meaningful: Use descriptive text like "John Smith, Project Manager" not just "avatar"
  • Context: Include role or context when relevant for identification
  • Concise: Keep alt text brief but informative
